关于静态代理的一些整理:
所谓代理:就是指a类需要调用c类的方法实现某个功能,
        但是a类无法直接调用c类的方法,这时需要一个b类作为代理类,
        a通过b类访问c类中的方法
这里我们将c类叫做目标类,b类叫做代理类,a类叫做客户类


举例:厂家,商家,消费者
注意:这里其实目标类和代理类的要做的事是一样的,就如同厂家和商家一样,都是为了出售某件商品,
     但是厂家是不会将商品直接卖给消费者的,消费者也无法直接想厂家购买,都是通过商家完成的,
     商家就是这其中的代理类
1:定义一个接口表示卖u盘的方法,表示厂家和商家在做的事
2:创建厂家类,实现步骤一
3:创建商家类,作为代理类,也需要实现步骤一的接口
4:创建客户类调用商家的方法购买u盘
